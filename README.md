# 🀄 麻将战绩排行榜系统

功能完善的麻将游戏战绩记录、统计和排行展示网站。

## 🚀 快速启动（Docker 一键运行）

```bash
docker-compose up --build -d
```

启动后访问 **http://localhost:8888**

## ✨ 主要功能

### 📊 排行榜系统
- **总排行榜**：胜率、总场次、总得分、场均、最大连胜/连败等多维度统计
- 支持按各指标正序/倒序排列
- 按时间范围（今日/本周/本月）筛选

### 📝 战绩录入
- **实时录入**：2-4名玩家，自动校验得分平衡（所有人合计=0）
- **历史补录**：与录入界面一致，自动标记"补录"标识
- **批量导入**：支持 Excel（.xlsx/.xls）和 CSV 格式，提供模板下载，支持预览和错误提示
- **牌型记录**：31种标准麻将牌型，支持自摸/点炮、番数、中码、连庄次数记录

### 🏆 特色榜单
- **全场通吃榜**：自动识别一人独赢场次，展示通吃王排行
- **反败为胜榜**：自动识别逆转场次，逆转走势可视化折线图，展示逆转王排行
- **高光时刻集锦**：自动评分，支持收藏和分享，多维度排序

### 📊 排行榜筛选
- 时间范围：今日 / 本周 / 本月 / **自定义日期区间**
- 游戏类型筛选（16张/13张/广东/四川等）
- 包含特定玩家筛选
- 最大连胜、**最大连败**列均支持排序

### 🏠 房间系统
- 创建游戏房间（支持房间码邀请）
- 访客可浏览，登录后可加入/创建房间
- 房间内一键录入战绩

### 👤 用户系统
- 注册/登录/访客浏览
- 头像上传（点击头像直接更换）
- 昵称/简介修改
- 个人主页（战绩统计 + 趋势图 + 对战历史）

### 📤 数据管理
- Excel / **PDF** 导出战绩
- **批量删除**战绩（列表页勾选后一键删除）
- **手动备份**（下载全量 JSON 备份文件）与**数据恢复**（从备份文件导入）
- 管理员可修改/删除任意战绩，普通用户只能操作自己的数据
- Django Admin 后台完整数据管理

## 🔧 本地开发

```bash
# 安装依赖
pip install -r requirements.txt

# 运行迁移
python manage.py migrate

# 初始化数据（管理员账号 + 牌型数据）
python manage.py init_data

# 运行开发服务器
python manage.py runserver
```

访问 http://127.0.0.1:8000

## 📊 数据导入格式

支持格式：**Excel（.xlsx / .xls）** 和 **CSV（.csv，UTF-8 编码）**

入口：战绩列表页 → 右上角「批量导入」按钮，可下载官方模板。

### 列结构

| 列 | 字段 | 必填 | 格式 / 说明 |
|----|------|:----:|-------------|
| A | 游戏时间 | ✅ | `YYYY-MM-DD HH:MM`，如 `2024-01-15 19:30` |
| B | 地点 | — | 任意文字，如 `家里`、`茶馆` |
| C | 游戏类型 | ✅ | 见下方可选值 |
| D | 底分 | ✅ | 正整数，如 `1`、`2` |
| E | 玩家1用户名 | ✅ | 系统注册的用户名（区分大小写） |
| F | 玩家1得分 | ✅ | 整数，可为负数 |
| G | 玩家2用户名 | ✅ | 同上 |
| H | 玩家2得分 | ✅ | 同上 |
| I | 玩家3用户名 | — | 可不填（2人局） |
| J | 玩家3得分 | — | 同上 |
| K | 玩家4用户名 | — | 可不填 |
| L | 玩家4得分 | — | 同上 |
| M | 备注 | — | 任意文字 |

### 游戏类型可选值

| 值 | 含义 |
|----|------|
| `mahjong_16` | 麻将16张（默认） |
| `mahjong_13` | 麻将13张 |
| `guangdong` | 广东麻将 |
| `sichuan` | 四川麻将 |
| `other` | 其他 |

### ⚠️ 注意事项

1. **用户名必须与系统账号完全一致**（填错会报"用户不存在"并跳过该行）
2. **所有玩家得分之和必须为 0**（如 `+30, -10, -10, -10`），否则该行报错跳过
3. 至少填写 **2 名玩家**
4. 第一行为表头，**从第二行开始填写数据**
5. CSV 文件请保存为 **UTF-8** 编码（Excel 另存为→CSV UTF-8）

## 故障排除

### 500 错误 / 数据库密码认证失败

若出现 `password authentication failed for user "mahjong_user"`，通常是 PostgreSQL 数据卷使用了旧的初始化数据。执行：

```bash
docker-compose down -v
docker-compose up -d
```

`-v` 会删除数据卷，PostgreSQL 会重新初始化并使用当前配置的密码。
